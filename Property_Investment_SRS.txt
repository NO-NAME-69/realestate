1. INTRODUCTION
1.1 Purpose
This SRS document specifies the complete requirements for the Property Investment Management System (RP
Module), which enables users to invest in property projects through a team-based investment model with wallet
management and profit distribution.
1.2 Scope
The system will manage:
• User registration and team building
• Investment wallet operations
• Property project and plot management
• Profit calculation and distribution
• Admin controls and reporting
1.3 Definitions, Acronyms, and Abbreviations
KYC: Know Your Customer
TSP: Town and Sector Planning
ERP: Enterprise Resource Planning
Admin: System Administrator
Team Value: Total investment amount from all team members
1.4 References
• User requirements document
• Business process flows
• Financial regulations for investment platforms
1.5 Overview
This document provides detailed functional and non-functional requirements organized by module. Each
module specification includes functional requirements with clear identifiers, input/output specifications, process
flows, validation rules, and business logic.
2. OVERALL DESCRIPTION
2.1 Product Perspective
A web-based investment management system enabling users to pool investments for property development
projects with automated profit distribution. The system operates as a standalone application with integrations to
third-party payment gateways, SMS/Email services, and cloud storage.
2.2 Product Functions
• User registration and team management (minimum 20 members per team)
• Wallet-based investment handling with ■500 minimum
• Property project lifecycle management
• Plot inventory and holding system
• Automated profit calculation and distribution
• Comprehensive reporting and analytics
2.3 User Classes and Characteristics
User Type Technical Expertise Primary Functions
Investor Low to Medium Register, invest, view returns
Team Leader Medium Manage team, track investments
Admin High System configuration, approvals, sales
2.4 Operating Environment
• Web-based application (responsive design)
• Mobile compatible (iOS and Android browsers)
• Cloud-hosted infrastructure
• Database: MySQL/PostgreSQL
• Backend: Node.js/Python/Java
• Frontend: React/Angular/Vue.js
2.5 Design and Implementation Constraints
• Payment gateway integration required for secure transactions
• Regulatory compliance for investment platforms (IT Act, RBI guidelines)
• Data encryption for sensitive information (PAN, bank details)
• Real-time transaction processing capability
• Audit trail maintenance for all financial operations
2.6 Assumptions and Dependencies
• Users have internet connectivity and basic devices
• Payment gateway services available 24/7
• Legal approvals obtained before system launch
• Users understand investment risks
• Company handles all legal/regulatory compliance
3. SYSTEM FEATURES AND REQUIREMENTS
This section details all functional requirements organized by module. Each requirement includes a unique
identifier, description, priority, process flow, inputs/outputs, and validation rules.
MODULE 1: USER MANAGEMENT MODULE
Purpose: Manage registration, activation, and user roles.
3.1.1 Functional Requirements
REQ-UM-001: User Registration
Description: System shall allow new users to register with required information
Priority: High
Input:
• Full Name
• Email Address
• Mobile Number
• Password
• Address
• PAN Number (optional)
Process:
1. User fills registration form
2. System validates all fields
3. System checks for duplicate email/mobile
4. Registration fee payment page displayed
5. User pays ■500 registration fee
6. System creates user account with 'Inactive' status
7. Confirmation email/SMS sent
Output: User account created with unique User ID
Validation Rules:
• Email must be valid format
• Mobile must be 10 digits
• Password minimum 8 characters with complexity
• All mandatory fields required
REQ-UM-002: Registration Fee Collection
Description: System shall collect ■500 registration fee
Priority: High
Payment Methods: UPI, Net Banking, Card, Wallet
Business Rule: Registration incomplete without payment
REQ-UM-003: User Profile Creation
System shall allow users to complete their profile with photo, DOB, occupation, bank details, and optional KYC
documents
REQ-UM-004: KYC Management
System shall support optional KYC verification with Aadhaar, PAN, Driving License, or Passport
REQ-UM-005: User Status Management
System shall maintain user status: Inactive (newly registered) or Active (team of 20 completed)
REQ-UM-006: Role Management
System shall assign roles: Investor (default), Team Leader, or Admin with appropriate permissions
REQ-UM-007: User Authentication
System shall authenticate users securely with email/mobile + password, session management, and account
lockout after 5 failed attempts
REQ-UM-008: Password Management
System shall allow password reset via OTP and password change with current password verification
REQ-UM-009: User Logout
System shall allow users to logout securely with session termination
MODULE 2: TEAM & NETWORK MANAGEMENT MODULE
Purpose: Build and track 20-member teams for user activation.
REQ-TM-001: Team Creation
System shall allow users to create a team with unique Team ID and referral link. One user can create only one
team.
REQ-TM-002: Team Member Invitation
System shall allow team leaders to invite members via referral link, email, SMS, or WhatsApp. System auto-fills
referrer information during registration.
REQ-TM-003: Team Size Validation
System shall validate minimum team size of 20 members. When count reaches 20, trigger activation logic.
REQ-TM-004: User Activation Logic
System shall activate users when team has 20 members. Status changes from Inactive to Active, enabling
investment features.
REQ-TM-005: Team Hierarchy Display
System shall display team structure with tree/network view showing members, join dates, status, and
investment amounts.
REQ-TM-006: Referral Tracking
System shall track all referral relationships storing referrer ID, referee ID, dates, and team assignments.
REQ-TM-007: Team Value Calculation
System shall calculate total team investment value as sum of all team members' investments in real-time.
REQ-TM-008: Team Performance Dashboard
System shall provide team performance metrics: total members, active/inactive, total investment, growth trend,
top performers.
REQ-TM-009: Team Member Management
System shall allow viewing and managing team members with search, filter, and profile access. Team leader
cannot remove members.
MODULE 3: WALLET & PAYMENT MANAGEMENT MODULE
Purpose: Handle all financial transactions with minimum ■500 investment and 24×7 payment availability.
REQ-WM-001: Wallet Creation
System shall auto-create wallet for each user upon registration with unique wallet ID and zero balance.
REQ-WM-002: Registration Fee Payment
System shall process ■500 registration fee via payment gateway with UPI, Card, Net Banking, or Wallet
options.
REQ-WM-003: Wallet Deposit/Top-up
System shall allow users to add money with minimum ■500 deposit and no maximum limit.
REQ-WM-004: Payment Modes and Frequency
System shall support Anytime (24×7), Weekly, and Monthly payment modes with scheduled transactions.
REQ-WM-005: Wallet Balance Display
System shall display current balance, total deposits, investments, withdrawals, and available balance in
real-time.
REQ-WM-006: Transaction History/Ledger
System shall maintain complete transaction history with ID, type, amount, status, timestamps, and balances.
Export to PDF/Excel.
REQ-WM-007: Payment Approval System
System shall support admin approval for payments above threshold. Auto-approval for smaller amounts.
REQ-WM-008: Investment from Wallet
System shall deduct investment amount from wallet, validating sufficient balance and minimum ■500
investment.
REQ-WM-009: Profit Credit to Wallet
System shall credit profit earnings to user wallet with transaction details and notifications.
REQ-WM-010: Payment Gateway Integration
System shall integrate with Razorpay/PayU/CCAvenue for PCI-DSS compliant transaction processing.
REQ-WM-011: Wallet Withdrawal
System shall allow withdrawal to registered bank account with minimum ■1000 and 3-5 business day
processing (optional feature).
REQ-WM-012: Transaction Security
System shall ensure secure processing with SSL/TLS encryption, 2FA for high-value transactions, and
comprehensive audit trail.
MODULE 4: INVESTMENT MANAGEMENT MODULE
Purpose: Control investment allocation based on user activation, team value, and wallet balance.
REQ-IM-001: Link Wallet to Investment
System shall link user wallet balance to investment functionality, transferring amount from wallet to investment.
REQ-IM-002: Investment Eligibility Check
System shall verify: User status Active, Team has 20+ members, Wallet balance ≥ ■500 before allowing
investment.
REQ-IM-003: Minimum Investment Validation
System shall enforce minimum investment of ■500 with no maximum limit.
REQ-IM-004: Investment Based on Team Value
System shall allow higher investment opportunities based on team value (e.g., premium plots for higher team
values).
REQ-IM-005: Investment Tagging - Project-wise
System shall tag investments to specific projects with Project ID, name, amount, date tracking.
REQ-IM-006: Investment Tagging - Plot-wise
System shall tag investments to specific plots. One investment can be split across multiple plots.
REQ-IM-007: Investment Amount Tracking Per User
System shall track total invested amount, active investments, project/plot breakdown, and history per user.
REQ-IM-008: Investment Amount Tracking Per Team
System shall track total team investment, average per member, top investors, project-wise allocation.
REQ-IM-009: Re-investment from Returns
System shall enable users to reinvest profit returns with full/partial amount options and auto-reinvest setting.
REQ-IM-010: Investment Confirmation and Receipt
System shall provide PDF receipt with investment ID, date, amount, project/plot details, emailed to user.
REQ-IM-011: Investment Portfolio View
System shall display complete portfolio with project/plot distribution, timeline, current value, expected returns.
REQ-IM-012: Investment Modification/Cancellation
System shall handle modifications with admin approval within policy timeframe (optional, may be disabled).
MODULE 5: PROPERTY & PROJECT MANAGEMENT MODULE
Purpose: Manage land, projects (Residential/Commercial/Resort/Hotel/Farmhouse), and development
lifecycle.
REQ-PM-001: Create Property Project
Admin creates projects with name, type, location, area, costs, completion date, photos. Project types: Land,
Resort, Hotel, Farmhouse.
REQ-PM-002: Large Land Purchase Entry
System records purchase details: seller info, area, price, survey numbers, legal documents, payment terms.
REQ-PM-003: Legal Approval Tracking - TSP
System tracks TSP approval: application date/number, status (Applied/Under Review/Approved/Rejected),
approval documents.
REQ-PM-004: Legal Approval Tracking - Government
System tracks: Environmental, Fire NOC, Water, Electricity, Building plan, Occupation certificate, RERA
registration.
REQ-PM-005: Plotting & Layout Management
System manages layout with blueprint upload, plot numbering, dimensions, types (Corner/Middle/Double-road),
pricing.
REQ-PM-006: Colony Development Status Tracking
System tracks phases: Land acquisition, Legal approvals, Planning, Infrastructure, Amenities, Handover
readiness.
REQ-PM-007: Project Cost Tracking
System tracks costs: Land purchase, Legal, Development, Construction, Marketing, Admin. Budget vs actual
analysis.
REQ-PM-008: Project Sales Tracking
System tracks: Total/sold/held/available plots, sales value, received/pending amounts, sale dates, revenue.
REQ-PM-009: Project Profitability Calculation
System calculates: Total Revenue - Total Cost = Gross Profit. Shows ROI%, investor share calculation.
REQ-PM-010: Project Status Management
System maintains status: Planned, Land Acquired, Legal in Progress, Approved, Under Development, Ready
for Sale, Completed, Closed.
REQ-PM-011: Project Gallery Management
System manages photos/videos with categories (Before/During/After), captions, featured image for
transparency.
REQ-PM-012: Project Reports
System generates: Project summary, Financial (cost/sales/profit), Development progress, Plot status,
Investor-wise investment, Timeline reports.
MODULE 6: PLOT INVENTORY & HOLDING MODULE
Purpose: Manage plots and temporary ownership (holding) based on team value and investment eligibility.
REQ-PH-001: Plot Creation
Admin creates plots with number, size, area, type (Corner/Middle/Double-road), facing, road width, price,
location, features.
REQ-PH-002: Plot Availability Status
System maintains status: Available, Held (temporarily), Sold (final). Color-coded display on listings and layout
map.
REQ-PH-003: Plot Holding by Users
Eligible users can hold plots temporarily. System records user ID, plot ID, hold dates, status with email/SMS
confirmation.
REQ-PH-004: Hold Eligibility - Team Value
User can hold plots worth up to 50% of team value (configurable). Example: ■10L team value → ■5L hold
capacity.
REQ-PH-005: Hold Eligibility - Investment Amount
User can hold plots worth up to 10x investment (configurable). System uses stricter of team value or investment
limit.
REQ-PH-006: Hold Validity & Duration
Default 30-day hold period (configurable). Daily job checks expiries, releases plots, notifies users 3 days before
expiry.
REQ-PH-007: Hold Lock Rules
Maximum X plots per user (e.g., 5). Only active users can hold. Cannot hold if outstanding dues. Specific error
messages.
REQ-PH-008: Plot Hold Release
Manual release by user, Auto release on expiry, Admin release. Released plots immediately available.
REQ-PH-009: Plot Comparison Feature
System allows 2-4 plot comparison: size, type, facing, road width, price, location advantages. Direct hold from
comparison.
REQ-PH-010: Plot Details Page
Display: Specifications, location map, photos, nearby amenities, status, legal info. Actions: Hold, watchlist,
download, share.
REQ-PH-011: Plot Search and Filter
Filter by: Project, type, size range, price range, facing, status. Search by plot number. Sort by price/size.
REQ-PH-012: Plot Availability Notification
Users add plots to watchlist. System notifies via SMS/Email/App when plot becomes available.
First-come-first-served.

Maintain complete sales history. Search/filter by project, buyer, date, status. Export sales summary and

MODULE 9: PROFIT CALCULATION & DISTRIBUTION MODULE
Purpose: Automate profit sharing with transparent calculations and wallet credits.
REQ-PC-001: Profit Calculation on Sale
Formula: Gross Profit = Sale Price - Allocated Cost. Triggered on plot sale confirmation. Can be positive or
negative.
REQ-PC-002: Company Percentage Deduction
Deduct company share (configurable %): Company Share = Gross Profit × %. Investor Pool = Gross Profit -
Company Share.
REQ-PC-003: Individual Investment-Based Calculation
User Profit = Investor Pool × (User Investment / Total Project Investment). Sum equals investor pool.
REQ-PC-004: Team Contribution-Based Calculation
Optional team bonus based on team value/size. Bonus = Base Share × Bonus %. Funded from company share.
REQ-PC-005: Profit Credit to Wallet
Credit calculated profit to wallets. Create transaction, update balance, set type 'Profit - Sale of Plot X', send
notification.
REQ-PC-006: Profit Notification
Send Email/SMS/In-app: Profit amount, project/plot details, investment, share %, transaction ID, new balance,
reinvest CTA.
REQ-PC-007: Profit Transparency Reports
User report: Plot sold details, sale price, cost, profit, company share, pool, your investment/share/profit.
Download PDF.
REQ-PC-008: Profit Calculation Audit Trail
Log: Timestamp, sale ID, input values, company %, calculated values, distribution, admin. For disputes and
compliance.
REQ-PC-009: Handle Loss Scenarios
If sale < cost: Options: Company absorbs loss, Pro-rata to investors, Offset future profits. Transparent
communication.
REQ-PC-010: Profit Distribution History
Maintain: Distribution ID, date, project/plot, total profit, investors count, per-investor amounts, status. User and
admin views.
REQ-PC-011: Profit Reinvestment Tracking
Track profits reinvested vs withdrawn. Link: Original profit source → Reinvestment destination. Show
compound growth.
REQ-PC-012: Bulk Profit Distribution
Handle multiple plot sales simultaneously. Aggregate profit by investor, single distribution, single notification
with breakdown.
MODULE 10: REINVESTMENT & REVENUE FLOW MODULE

Users invest in newly launched projects. Notify profitable investors. Track investment separately.
REQ-RV-004: Manual Reinvestment Option
User decides: Reinvest (browse projects/plots, invest) or Withdraw (transfer to bank). Profit stays in wallet until

REQ-RV-005: Auto Reinvestment Option
Optional auto-reinvest setting. User sets: Minimum amount, preferred project/plot types. System auto-invests

REQ-RV-006: Revenue Tracking Project-wise
Track per project: Total investment, plots sold, sales revenue, profit, distributions, company share, unsold plots,
projected revenue.
REQ-RV-007: Revenue Cycle Tracking
Track full cycle: Initial investment → Allocation → Plot sale → Profit distribution → Reinvestment → Repeat.
Show money flow visualization.
REQ-RV-008: Team Revenue Reports
Team leader reports: Total team investment/profit, Average ROI, Top contributors, Growth trend, Reinvestment
vs withdrawal ratio.
REQ-RV-009: Reinvestment Incentives Tracking
Track incentives: Bonus interest, Priority plot access, Reduced fees, Loyalty rewards. Apply and record when
user reinvests.
REQ-RV-010: Cash Flow Management
Admin dashboard: Inflows (deposits, sales), Outflows (land, development, distributions), Net position, Trends.
Alert if negative/low.
REQ-RV-011: Revenue Projection
Project future revenue: Unsold plots × prices × expected sale probability. Admin dashboard with disclaimer. For
business planning.

MODULE 13: SECURITY & COMPLIANCE MODULE
Purpose: Ensure system safety with encryption, authentication, and regulatory compliance.
REQ-SC-001: Role-Based Access Control
Define roles and permissions. Enforce checks on all actions. Backend validates, frontend hides unauthorized
actions. Log unauthorized attempts.
REQ-SC-002: Transaction Security
HTTPS/SSL, PCI-DSS compliance, tokenization, no full card storage, transaction signing, replay prevention,
session timeout, 2FA for high-value, fraud detection.
REQ-SC-003: Data Encryption
In Transit: TLS 1.2+. At Rest: Passwords (bcrypt/Argon2), PAN/Aadhaar/Bank (AES-256). Secure key
management, rotation policy.
REQ-SC-004: Audit Trail
Comprehensive logging: All CRUD on sensitive data, failed logins, IP/user agent. Tamper-proof append-only
logs. 3+ year retention.
REQ-SC-005: Authentication & Password Security
Min 8 chars with complexity, strength indicator, prevent weak passwords. Lockout after 5 failures (30 min).
CAPTCHA after 3. Secure reset with OTP.
REQ-SC-006: Session Management
Secure tokens, HttpOnly cookies, 30-min inactivity timeout, 12-hr absolute timeout. Logout invalidates. Prevent
hijacking: bind to IP, regenerate on login.
REQ-SC-007: Input Validation & Sanitization
Frontend and mandatory backend validation. Whitelist approach, type/range/format checking. Prevent SQL
injection (parameterized queries), XSS (escape output, CSP), command injection.
REQ-SC-008: Legal Compliance Support
